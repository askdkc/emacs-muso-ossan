* まえがき

これは、Emacs を知っている人向けの物語ではない。

むしろ逆だ。

Git の `add` と `commit` と `branch` の違いで指が止まる人。Docker の赤いログを見ると、心臓が一瞬止まる人。Slack の通知が鳴るたび、何かを責められているように感じる人。Notion に綺麗なメモを書こうとして、結局何も書けなくなる人。ターミナルにコマンドを打つたび「これで壊れないよね？」と不安になる人。

そういう人のための話だ。

主人公は天城ミオ。二十三歳。小悪魔みたいな笑みを浮かべる、少しギャルっぽい新人エンジニア。

見た目は強そうだ。口も軽い。先輩にもわりと遠慮なく突っ込む。

でも中身は、普通に焦る。普通に怖い。普通に分からない。普通に「わたし、この仕事向いてないかも」と思う。

彼女が配属されたのは、月額課金型のタスク管理サービスを作る開発チーム。

Next.js。TypeScript。Node.js。PostgreSQL。Redis。Docker。GitHub Actions。Slack。Sentry。Datadog。

今風のスタックが並んでいる。だが現場は、そんなに綺麗ではない。

通知は飛ぶ。PR は溜まる。テストは落ちる。DB は遅い。Docker は起動しない。手順書は古い。本番は金曜の夜に燃える。

そこに現れるのが、牧野源次郎。

黒いシャツ。白髪まじりの髪。眼鏡。少し無精ひげ。社内で「オッサン無双エンジニア」と呼ばれている男。

彼の武器は、伝説のツール Emacs。

ただしこの物語は、「古いエディタすごい」と言うだけの話ではない。

Emacs を使うと、なぜ仕事が速くなるのか。Magit を使うと、なぜ Git が怖くなくなるのか。org-mode を使うと、なぜタスクとメモとコードと結果が一つの流れになるのか。文学プログラミングを使うと、なぜ手順書が「読まれない紙」から「実行できる説明」になるのか。

その気持ちよさを、物語として書く。

* PR #421 review
** 観察
- ログイン後の redirect 処理が二重に走っている
- session fetch が Server Component と Client Component の両方にある
- loading 表示の終了条件が曖昧

** 仮説
- duplicate fetch によりログイン後の表示が遅く見えている

** 提案
- session fetch を server side に寄せる
- client には必要最小限の user state だけ渡す
#+end_src

* あとがき

この改訂版では、初心者が「何に困っているのか」を厚めに描いた。

特に Magit については、単に「速い」とは書かないようにした。

コマンドラインで Git を使うと、初心者は何度も迷う。

- `git switch -c` だっけ、`git checkout -b` だっけ
- `git add .` で全部入れていいのか
- `git commit -am` は untracked file を含まないのか
- push 先 upstream はどうするのか
- 今 staged なのか unstaged なのか
- branch はどこなのか

Magit は、それらを画面上の状態として見せる。

そして、

#+begin_example
b b  branch
s    stage
u    unstage
c c  commit
P p  push
F p  pull
l l  log
d d  diff
#+end_example

のように、キーボード操作で進められる。

速い理由は、一文字で操作できるからだけではない。

迷う時間が減るから速い。状態が見えるから速い。長いコマンドを手で打たなくていいから速い。commit message を Emacs のバッファで落ち着いて書けるから速い。

それが、初心者にとっての Magit の強さだ。

また、org-mode についても、単なる TODO 管理ではなく、以下の役割を物語に入れた。

- 頭の中の混乱を Inbox に出す
- TODO に状態を付ける
- agenda で今日を見る
- エラー調査を「事象・仮説・確認・結論」にする
- org-table でデータを整える
- org-babel でコードや SQL や shell を実行する
- 文学プログラミングで手順書を生きた文書にする
- incident 対応をバッファ内で構造化する

Emacs の魅力は、全部を Emacs だけでやることではない。

Slack、GitHub、Notion、Docker、PostgreSQL、Redis、TypeScript、GitHub Actions。そうした現代的なツールの真ん中に、自分だけの作業台を作ることだ。

空のバッファに、まずこう書く。

#+begin_src org
* Inbox
#+end_src

そこから、世界は少しずつ整っていく。
